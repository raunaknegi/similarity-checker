Document Similarity API Service

The purpose of this service is to return the similarity score of 2 documents based on the model chosen by user. The API is build via Flask and deployed on heroku.

In order to use the API, simply send a post request to either of the two urls:

  1. /predict_infersent
  2. /predict_others

Two different urls are used because InferSent's model building is very different from the rest of the models. In case of using the InferSent model, the post request should contain the 2 documents as a json object, with the key names "doc1" and "doc2". Moreover, InferSent on its first request trains and builds the vocab, so it will take some time before responding the score. It is necessary that the 2 values are strings, as other data types create complexity in carrying out the tasks.

For using the "/predict_others" URL, the json object should contain the following 3 keys:

  1. "doc1":string
  2. "doc2":string
  3. "modelName":string
The following model names should be used for predicting the similarity via this route (case sensitive):
  • BERT
  • jaccard_similarity
  • tf_idf
  • USE
Please wait for the pretrained models(BERT and USE) to download. Once downloaded, the response time will be much quicker.
